# 代码文件位置规范

规则类型: 强制规则
强制级别: 严格执行

---

## 核心原则

### 所有代码文件必须在project-code/目录

绝对禁止:
- 在项目根目录创建.py文件
- 在项目根目录创建脚本
- run_app.py start.py等放在根目录

必须执行:
- 所有Python代码放在 project-code/
- 启动应用直接运行 project-code/app.py
- 打包脚本: cd project-code && python app.py

---

## 运行方式

正确:
cd project-code
python app.py

错误:
python run_app.py  # 根目录禁止此类文件

---

## 根目录允许的文件

- README.md
- .gitignore
- requirements.txt
- project-config.md
- LICENSE

其他所有文件必须在对应子目录中!
