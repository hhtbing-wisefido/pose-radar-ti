## 📝 记录文档命名规范

### 格式要求
```
YYYY-MM-DD_[类型]_[简短描述].md
```

### 类型标签
- `问题排查` - 问题诊断和解决过程
- `修复计划` - 问题修复的执行计划
- `分析报告` - 技术分析和对比报告
- `开发笔记` - 开发过程中的临时笔记
- `配置说明` - 配置变更说明

### 示例
```
✅ 2025-12-15_问题排查_串口通信异常.md
✅ 2025-12-15_修复计划_固件烧录流程.md
✅ 2025-12-15_分析报告_SDK文档对比.md
```

## 🗂️ 归档规则

### 保留期限
- **活跃记录**: 近3个月的记录保持在此目录
- **历史记录**: 3个月以上的记录可归档到`历史记录/`

### 清理标准
- 临时调试记录：问题解决后可删除
- 重要决策记录：永久保留
- 重复/过时内容：及时清理
